# WormVision Smoke Test Checklist

发版前接相机走一遍。每条都是可勾选项。**任何一条不过都不能发版**。

> 自动化测试（ctest）已经覆盖**纯逻辑层**（VideoUtils、DatabaseManager、
> RecordingDiagnostics、VideoLibraryService 共 5 个 suite / 51 个 case）。
> 本文档专门覆盖 GUI + SDK + 硬件层。

---

## 环境前置

- [ ] Windows 10/11 x64，已安装 **海康 MVS 客户端**（提供相机驱动）
- [ ] 海康相机已连接（GigE / USB3.0），用 MVS 客户端能识别且预览正常
- [ ] WormVision 已通过 `WormVision-Setup-X.Y.Z.exe` 安装

---

## 1. 启动 & 基本 UI

- [ ] 双击桌面快捷方式或 `WormVision.exe`，**5 秒内**主窗口出现
- [ ] 窗口标题：`WormVision - 工业相机采集系统`
- [ ] 工具栏 3 个按钮：采集 / 视频库 / 切换主题
- [ ] 默认在"采集"视图

## 2. 设备识别（中文路径/型号兼容）

- [ ] 点"刷新设备列表"，下拉框出现至少 1 个相机
- [ ] 设备名称**不出现 `?` 或乱码**（验证 Phase 3 `fromLocal8Bit` 修复）
- [ ] 序列号显示正常
- [ ] 状态栏显示"相机已连接 (就绪)"

## 3. 参数面板

- [ ] **曝光/增益/帧率**滑块和数值都不是 0（验证 SDK 读取了真实范围）
- [ ] 拉动曝光滑块，**预览图像实时变亮/变暗**
- [ ] 拉动增益滑块，预览即时响应
- [ ] 分辨率显示"宽 x 高"，与相机型号匹配
- [ ] 最大分辨率标签显示

## 4. 预览

- [ ] 点"开始预览"，**1 秒内**视频流出现
- [ ] 状态栏 FPS **持续刷新**（不是 0、不冻结）
- [ ] 工具栏显示`分辨率：W x H`、`帧数：递增`
- [ ] 缩放按钮 `-` / `+` / `适应` 正常工作
- [ ] 鼠标滚轮在画面上能缩放
- [ ] 点"停止预览"后画面变黑

## 5. 抓拍

- [ ] 预览中点"抓拍"，状态栏无错误
- [ ] `build\snapshots\` 出现 `snapshot_yyyyMMdd_HHmmss.jpg`
- [ ] **文件大小 > 0** 且能在系统图片查看器打开
- [ ] 在"任务名称"输入框输入中文（如`线虫_实验1`）后抓拍 → 文件名含中文

## 6. 录制（核心防回归）

- [ ] 输入任务名 `test_record` → 点"开始预览" → 点"开始录制"
- [ ] 状态栏出现红色 `● 录制中 00:0X`，计时正常增加（每秒只 +1，不是 +2 —— 验证 Phase 3 重复 connect 修复）
- [ ] 录 **10 秒** 后点"停止录制"
- [ ] **不弹"录制问题"对话框**（验证 Phase 5 FFmpeg DLL + 像素转换修复）

## 7. 录制完成后入库（Phase 4 防回归）

- [ ] 录制停止后**无需任何操作**，切到"视频库"tab
- [ ] 新录的视频**直接出现**在列表里（不需要手动刷新）
- [ ] **时长显示约 0:10**（不是 `--:--` —— 验证 Phase 1 时长解析 + Phase 4 自动入库）
- [ ] **文件大小** > 几 MB（不是 0 B —— 验证 Phase 5 录制 0 字节修复）
- [ ] 双击文件能在 WMP / VLC / 系统播放器**播放出图像**（不报 0xc10100be）

## 8. 视频库管理

- [ ] 右键视频 → "重命名" → 输入新名（含中文）→ 文件被改名 + DB 同步
- [ ] 右键 → "打开所在文件夹" → 系统资源管理器跳到 `recordings/`
- [ ] 右键 → "上传到云端 (Mock)" → 弹出"功能尚未实现"提示（**不再说"上传成功"** —— 验证 Phase 4 修复）
- [ ] 勾选 1 个 → "删除选中" → 文件 + DB 记录都消失

## 9. 脏数据清理（Phase 5 防回归）

- [ ] 在 `recordings\` 手动 put 一个 0 字节假文件（如 `fake.avi`）
- [ ] 在 `recordings\` 手动删一个真实视频但 DB 还记着的（手动改名）
- [ ] 重启 App，进视频库
- [ ] 0 字节文件**自动从文件系统消失**，DB 列表不显示它
- [ ] 不存在文件的 DB 记录被清理，不显示

## 10. 主题切换

- [ ] 工具栏点"切换主题" → 整体配色切到亮色
- [ ] 再点一次 → 切回暗色
- [ ] 切换不闪烁不崩

## 11. 关闭

- [ ] 点窗口右上角 X
- [ ] 任务管理器无 `WormVision.exe` 残留
- [ ] **没有 mini-dump 文件**生成

## 12. 日志

- [ ] `build\wormvision.log`（或装好版本里 `<安装目录>\wormvision.log`）存在
- [ ] 文件以 `==================== WormVision 启动 ====================` 开头
- [ ] 录制段落能看到 `录制像素类型: XXX → BGR8 (需要转换)` 类型的诊断行
- [ ] 没有 `[E]` 或 `[F]` 级别的错误（warning `[W]` 可以接受）

---

## 故障速查

| 现象 | 大概率原因 | 第一步排查 |
|---|---|---|
| 设备列表空 | MVS 驱动没装 / 相机断电 | 打开 MVS 客户端能不能见到 |
| 预览黑屏 | `MvRender.dll` 缺失或网卡 jumbo frame 没开 | 看 `wormvision.log` 有无加载错误 |
| 录制 0 字节 | FFmpeg DLL 缺（部署没拷 ThirdParty） | `<安装目录>\swscale-9.dll` 是否存在 |
| 设备名乱码 | 老版本 `fromLatin1` 没修 | 看 commit 是否包含 Phase 3 |
| 库时长 `--:--` | mp4 时长解析失败 / 文件损坏 | 用 `VideoUtils::parseVideoDurationFromFile` 单测验证 |
